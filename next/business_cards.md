# Business Cards

Printable business cards generated by Praxis itself: gear-like, fractal
geometry sampled at runtime, cropped and zoomed in interesting places, themed
to match the web app. Front and back PDFs people can print at home, plus an
A4 imposition sheet. This doubles as the first public demo of the "suite of
functions that modulate 2D projections" we want for the spider roadmap - the
card is just a tiny viewport into a procedural field.

## Standard

- Card size: 85 x 55 mm (EU standard). It tiles A4 cleanly as 2 columns x 4
  rows = exactly 8 cards per sheet, which matches the "download 8" button and
  common print-at-home templates (Avery-style, crop marks at cell corners).
- Two single-page PDFs per download: `front.pdf` and `back.pdf` (single card,
  centered on A4 with crop marks), or `sheet-front.pdf` / `sheet-back.pdf`
  (8-up). The back sheet mirrors columns horizontally so long-edge duplex
  printing aligns front to back. No rotation on either side.
- 3 mm bleed-free inset border: white in light theme, black in dark theme.
  The artwork stops at the border; cleaner than full-bleed on home printers.

## Geometry: a projection registry

The generative core lives outside the web app, in a new
`praxis/pillars/projections.py` (or `praxis/projections/` if it grows):

- `PROJECTION_REGISTRY`: named functions `(seed, width, height) -> field`,
  each a pure 2D scalar/vector field. Initial entries: `gear` (polar tooth
  profiles, nested sub-gears via recursion), `fractal` (escape-time or IFS,
  low iteration count for "simple detail"), maybe `lissajous`. Registry
  entry, not CLI flags, per house style.
- A `Viewport` sampler: given a seed, pick a projection, a crop window
  (zoom 2x to 8x into an off-center region), and a stroke density. The crop
  is what makes cards look "focused/cut in interesting places" instead of a
  centered mandala.
- Everything is seeded; the same seed reproduces the same card. The preview
  endpoint hands the seed back so download matches what was rendered.

Rendering uses matplotlib (already a dependency, Agg backend, same pattern as
`praxis/pillars/evolution.py`). Vector output: lines and polygons drawn as
matplotlib artists, saved as SVG for the in-place preview and PDF for
downloads. No new dependencies - reportlab/cairo rejected, matplotlib writes
both formats natively and stays vector.

## Color

- Theme comes from the client as query params: `theme=light|dark` and
  `hue=161` (the `--accent-hue` CSS variable, read at request time so an
  accent change follows automatically).
- Palette: paper background (#f9f9f9 / #1a1a1a to match the app), geometry
  stroked in 2 or 3 tints of the accent hue at low-to-mid saturation, one
  near-neutral. Simple and non-obtrusive; the geometry carries the card.

## Text

- Front: names from `--author` (already parsed, list, default
  "Ryan J. Brooks"; available via `current_app.config` like `donations`).
- Back: the `--donations` URL, plus the run's truncated hash as a small
  footer mark (it is the Identity tab, after all).
- Capitalization is sampled per render from the seed: each text element
  independently picks from {Title Case, UPPER, lower, SMALL CAPS via sizing}.
  Different sampling distributions for front vs back.
- Fonts: matplotlib's bundled DejaVu, monospace for the URL/hash to echo the
  dashboard's code blocks.

## Server

New blueprint `praxis/web/routes/cards.py`, registered in
`routes/__init__.py`:

- `GET /api/card/preview.svg?seed=&theme=&hue=` - inline preview (front).
  `side=back` for the back. Fresh seed when none given; response includes the
  seed in a header so the client can pin it.
- `GET /api/card/front.pdf` and `/api/card/back.pdf` - single card on A4 with
  crop marks. Same query params; `seed` required so it matches the preview.
- `GET /api/card/sheet-front.pdf` and `/api/card/sheet-back.pdf` - 8-up A4.
  Eight distinct seeds derived from the base seed (eight different crops of
  the field is a better demo than eight copies), back sheet column-mirrored.

Rendering a card is milliseconds of CPU and touches no model state, so this
is a plain on-demand route - no SnapshotStore producer, no eval-race surface.
`send_file` with proper download names, mirroring `/api/paper.pdf`.

## Web app

Edit `praxis/web/src/` only, then `python praxis/web/src/build.py`:

- New section in `renderSpecSections` in `tabs.js`: "Business Card". Renders
  the front preview in place by default via `<img src="/api/card/preview.svg?...">`,
  passing the current theme and the live `--accent-hue` value. A small
  "flip" affordance to show the back is nice-to-have, not required.
- Buttons beside it: "Download" (fetches front.pdf and back.pdf with the
  pinned seed) and "Download 8" (the two sheet PDFs). A "reroll" control to
  resample the seed.
- Styling stays in `src/css/`, reusing spec-section chrome. The preview gets
  the theme-appropriate border so it looks like the printed card.

## Phasing

Status: phases 1-3 shipped (POC). Margin note on PDFs tells people what
cardstock to buy and points at Amazon for pre-perforated 85 x 55 sheets.

1. `projections.py` with `gear` + viewport sampler + seeded text styling;
   unit test that a fixed seed renders deterministic SVG/PDF bytes.
2. `cards.py` routes (preview + 2 single PDFs), Identity tab card with
   Download button.
3. 8-up imposition with mirrored back, Download 8 button.
4. More registry entries (`fractal`, others) as the projection suite grows
   toward the spider work; the card endpoint needs no changes to benefit.
